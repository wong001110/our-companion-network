# S3 public Companion contract

A Network Companion is a separate server-generated ID. It contains only `id`, owner ID, approved name, optional public description, normalized tags, `friends_only` visibility, published state and active Asset Pack ID. A local Companion ID, assets paths, memories, prompts, personality and account email never enter this contract. A user has zero or one active Network Companion; only accepted, unblocked friends can read its published active profile.
